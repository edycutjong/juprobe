# Juprobe — Build Plan (3-Day Sprint)

## Day 1: Core Integration (8 hours)
Jupiter API integration (6 endpoints), stress test runner, latency measurement

## Day 2: Features + Polish (8 hours)
Charts (Recharts), error taxonomy, DX report generator

## Day 3: Demo + Submit (8 hours)
Polish dashboard, generate 2,000-word report, demo video

## Must-Have ✅
- Multi-endpoint stress tester (Quote, Swap, Limit Order, DCA, Token List, Price)
- Latency charts (p50/p95 per endpoint)
- Error taxonomy (categorize failure modes)

## Nice-to-Have 🟡
- Auto-generated DX report (2,000 words)
- Exportable benchmark data (JSON + PDF)
